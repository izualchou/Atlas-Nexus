# Chapter 01 - Project Overview

## 1.1 Purpose
Atlas Nexus is an AI-driven Tasker XML engineering platform. Its objective is to transform natural language requirements into deterministic, production-ready Tasker XML through metadata-driven reasoning, validated generation, and round-trip verification.

## 1.2 Vision
Natural Language -> Metadata Analysis -> XML Generation -> Round-trip Validation -> Production Deployment.

## 1.3 Scope
Includes Task XML, Profile XML, Project XML, Tasker Metadata, Java Function, AutoInput, SQLite, HTTP Request, network verification, and AI-assisted XML generation.

Excludes beginner Tasker tutorials, Android ROM modification, and root-specific development.

## 1.4 Goals
- Deterministic XML generation
- Metadata-first engineering
- Round-trip validation
- Incremental deployment
- AI-first workflow

## 1.5 Non-Goals
- Generic Android programming guide
- Beginner automation tutorial
- Root-only framework

## 1.6 Design Philosophy
XML-first, Metadata-first, Validation-first, Incremental-first, Reusable-first, Deterministic-first.

## 1.7 Engineering Workflow
Metadata -> Action -> Template -> Pattern -> Primitive -> Engine -> Runtime -> Generator -> Validation -> Deployment.

## 1.8 Architecture Overview
Atlas Nexus is organized into Foundation, XML Language, Runtime, Java Framework, Network Verification, Recovery Framework, Generator Specification, AI Specification, Deployment, and Reference.

## 1.9 AI Generator Pipeline
Requirement -> Metadata Analysis -> Capability Selection -> Action Planning -> Pattern Composition -> Engine Composition -> XML Generation -> Round-trip Validation -> Deployment.

## 1.10 Document Structure
The PART I–PART X structure is frozen. Future revisions expand content without restructuring the document.
